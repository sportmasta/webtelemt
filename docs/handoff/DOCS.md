# WebTelemt — руководство: покупка профиля и настройка оплаты

Документ для **покупателей** и **администраторов** сервера. Технические детали реализации — в [`DEV.md`](DEV.md).

---

## Для покупателя: как купить профиль

### Шаг 1. Откройте страницу покупки

Перейдите по ссылке, которую дал администратор, например:

`https://ваш-сервер.example/buy`

На странице отображаются название тарифа, цена и срок (информационно).

### Шаг 2. Заполните форму (по желанию)

| Поле | Обязательно | Описание |
|------|-------------|----------|
| Имя пользователя | Нет | Латиница, цифры, `_`, `.`, `-` (1–64 символа). Если пусто — система сгенерирует имя вида `user_xxxxx` |
| Email | Нет | Сохраняется в заказе; письмо с данными **не отправляется** |

### Шаг 3. Оплатите

Нажмите **«Оплатить»** — откроется защищённая страница ЮKassa. Выберите способ оплаты и завершите платёж.

### Шаг 4. Сохраните учётные данные

После успешной оплаты вы попадёте на страницу **«Оплата прошла успешно»** (`/buy/success?order_id=...`).

На ней **один раз** показываются:

- **Имя** (username) — для подключения к прокси
- **Секрет** — пароль профиля

**Сохраните их сразу** (скриншот, менеджер паролей). Повторно получить секрет через сайт нельзя.

Если страница пишет «Оплата обрабатывается» — подождите несколько секунд и обновите страницу (webhook может идти с небольшой задержкой).

### Если оплата не прошла

При отмене или ошибке откроется `/buy/fail`. Средства не списаны — можно вернуться на `/buy` и попробовать снова.

### Подключение к Telegram-прокси

Используйте выданные имя и секрет в клиенте Telegram (MTProto proxy) по инструкции вашего провайдера. Администратор панели может подсказать формат ссылки `tg://proxy`.

## Для администратора: настройка оплаты

### Предварительные условия

1. WebTelemt установлен (`./install.sh`), Telemt API доступен на `127.0.0.1:9091`.
2. Контейнеры запущены: `cd development && docker compose up -d` (поднимаются **webtelemt** и **postgres**).
3. Есть аккаунт [ЮKassa](https://yookassa.ru/) с подключённым магазином.

### 1. Переменные в `development/.env`

После установки уже заполнены `DATABASE_URL`, `BILLING_CREDENTIALS_ENCRYPTION_KEY`, тариф и `YOOKASSA_RETURN_URL`. **Обязательно укажите:**

```env
YOOKASSA_SHOP_ID=<ID магазина>
YOOKASSA_SECRET_KEY=<секретный ключ>
```

При необходимости измените тариф:

```env
BILLING_PLAN_NAME=Базовый
BILLING_PLAN_PRICE_RUB=299
BILLING_PLAN_PERIOD_DAYS=30
```

Для production задайте публичный URL возврата (без пути — `order_id` добавит backend):

```env
YOOKASSA_RETURN_URL=https://ваш-домен.example
```

Перезапуск после правок:

```bash
cd development && docker compose up -d
```

### 2. Webhook в ЮKassa

1. Войдите в [личный кабинет ЮKassa](https://yookassa.ru/my).
2. **Настройки** → **Уведомления** (HTTP-уведомления).
3. URL: `https://ваш-домен.example/api/billing/webhook/yookassa`
4. Включите событие **`payment.succeeded`**.

Требования:

- Публичный **HTTPS**-адрес (reverse proxy: nginx, Caddy и т.п.).
- Порт панели по умолчанию — `8080` (`PANEL_PORT` в `.env`).

Backend при webhook **повторно запрашивает статус платежа** в API ЮKassa и сверяет сумму с заказом — это защита от поддельных уведомлений.

### 3. Проверка

| Проверка | Ожидание |
|----------|----------|
| `http://<сервер>:8080/buy` | Страница с тарифом и кнопкой оплаты |
| Тестовый платёж (режим ЮKassa) | Редирект на success, показ username и secret |
| Вход в панель → вкладка **«Заказы»** | Заказ со статусом `completed`, username, без secret |
| Повторное открытие success-страницы | Сообщение, что данные уже были показаны |

### 4. Админ-панель: заказы

После входа в панель откройте вкладку **«Заказы»**. Для каждого заказа видны:

- ID, дата, сумма, статус
- Запрошенное и выданное имя пользователя
- Email (если покупатель указал)
- Флаг, были ли учётные данные просмотрены покупателем

**Секрет в админке не отображается** — только факт выдачи.

### 5. Удаление и данные

```bash
./install.sh --uninstall              # остановить контейнеры
./install.sh --uninstall --purge-postgres   # + удалить volume PostgreSQL (все заказы)
./install.sh --uninstall --purge      # + удалить .env
```

## Статусы заказа

| Статус | Значение |
|--------|----------|
| `pending` | Заказ создан, ожидает оплаты |
| `paid` | Оплата подтверждена, создаётся профиль |
| `completed` | Профиль создан, secret можно получить один раз |
| `failed` | Ошибка (платёж или создание профиля) |
| `expired` | Не оплачен в течение TTL (по умолчанию 60 мин) |

## FAQ

### Покупатель не видит секрет после оплаты

1. Обновите страницу `/buy/success?order_id=...` через 10–30 секунд.
2. Убедитесь, что webhook ЮKassa настроен и доступен с интернета по HTTPS.
3. Проверьте логи: `docker compose -f development/docker-compose.yml logs -f webtelemt`.
4. В админке найдите заказ: если `failed` — смотрите доступность Telemt API.

### «Учётные данные уже были показаны»

Секрет выдаётся **один раз**. Повторный запрос невозможен (by design). Администратор может создать нового пользователя вручную в панели или оформить новый заказ.

### Страница `/buy` пишет «Тариф недоступен»

Не заданы переменные биллинга в `.env` или контейнер не перезапущен. Нужны: `DATABASE_URL`, `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY`, `YOOKASSA_RETURN_URL`, `BILLING_CREDENTIALS_ENCRYPTION_KEY`.

### Можно ли изменить цену без пересборки?

Да — измените `BILLING_PLAN_PRICE_RUB` в `.env` и перезапустите `docker compose up -d`. Редактирования тарифов через UI нет.

### Отправляется ли email с паролем?

Нет (фаза 2). Поле email только сохраняется в заказе для учёта.

### Нужен ли отдельный личный кабинет покупателя?

Нет. Покупатель получает данные на success-странице после оплаты. Повторного входа по заказу нет.

### Безопасность

- Rate limit на создание заказов и вход в панель.
- Secret в БД шифруется (Fernet); после первого просмотра запись удаляется.
- Secret и ключи ЮKassa не пишутся в логи.
