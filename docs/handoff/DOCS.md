# WebTelemt — руководство: покупка, личный кабинет и настройка оплаты

Документ для **покупателей** и **администраторов** сервера. Детали реализации — в [`DEV.md`](DEV.md).

**Фаза 3 (2026-06-24):** личный кабинет покупателя — регистрация, вход, история заказов, live-профили Telemt (без secret).

---

## Страницы для покупателя

| URL | Назначение |
|-----|------------|
| `/buy` | Покупка профиля |
| `/buy/success?order_id=...` | Однократная выдача username и secret |
| `/buy/fail` | Отмена или ошибка оплаты |
| `/account` | Личный кабинет (требует вход) |
| `/account/login` | Вход |
| `/account/register` | Регистрация |

Пример: `https://ваш-сервер.example/buy`, `https://ваш-сервер.example/account`

## Личный кабинет: регистрация и вход

### Регистрация

1. Откройте `/account/register`.
2. Укажите **email** и **пароль** (минимум 8 символов), подтвердите пароль.
3. После успешной регистрации вы попадёте в кабинет `/account`.

При регистрации или входе система **автоматически привязывает** старые заказы с тем же email (см. ниже).

### Вход

1. Откройте `/account/login`.
2. Введите email и пароль.
3. Сессия сохраняется в браузере (JWT в `localStorage`, ключ `customer_token`).

Ссылки «Личный кабинет» и «Регистрация» есть на странице `/buy` и после успешной оплаты.

### Что не реализовано (фаза 3)

- Подтверждение email, восстановление пароля.
- OAuth (Google, Telegram).
- Повторная выдача secret через кабинет.

## Что видно в кабинете (`/account`)

### Шапка

- Email аккаунта.
- **«Купить ещё»** → `/buy`.
- **«Выйти»** — выход из кабинета.

### Блок «Мои профили»

По каждому **завершённому** заказу с выданным username:

- имя пользователя, дата выдачи, статус заказа;
- live-данные из Telemt: статус, TCP-сессии, активные IP, трафик;
- **ссылки подключения** (`tg://proxy`) и QR-код — **без secret**;
- пометка «секрет просмотрен», если данные уже показывались на success-странице.

Данные Telemt обновляются **каждые 10 секунд**.

### Блок «История заказов»

Все заказы аккаунта: дата, сумма, статус, выданный username.

## Покупка с привязкой к аккаунту

### Если вы вошли в кабинет

- Заказ автоматически привязывается к аккаунту.
- Email подставляется из профиля (поле только для чтения).

### Если не вошли

- Поле **email обязательно** — без него заказ не создаётся.
- Email сохраняется в заказе для последующей привязки к кабинету.

### После оплаты

На `/buy/success` **один раз** показываются username и secret — сохраните их сразу.

Если вы **не залогинены**, но указали email, появится призыв **«Создать кабинет»** / **«Войти»** — чтобы позже видеть профиль и ссылки подключения (без secret).

---

## Привязка старых заказов по email

Если вы покупали **до регистрации** и указывали email при оплате:

1. Зарегистрируйтесь или войдите с **тем же email** (регистр не важен — `user@mail.com` = `User@Mail.com`).
2. Все заказы с этим email, у которых ещё не было привязки к аккаунту, появятся в кабинете.

**Важно:** если при покупке email не указывали, постфактум привязать заказ нельзя.

## Как купить профиль (кратко)

1. `/buy` — тариф, форма (username по желанию, email — см. выше).
2. **«Оплатить»** → страница ЮKassa.
3. После оплаты — `/buy/success`, сохраните secret.
4. Для доступа позже — кабинет `/account`.

Подключение: используйте username и secret в клиенте Telegram (MTProto proxy) или ссылки из кабинета.

## Личный кабинет vs админ-панель

| | Личный кабинет | Админ-панель |
|---|----------------|--------------|
| **URL** | `/account`, `/account/login` | `/` (корень) |
| **Кто** | Покупатель | Администратор сервера |
| **Вход** | Email + пароль (регистрация) | Логин/пароль из `.env` (`PANEL_ADMIN_*`) |
| **Токен** | `customer_token` | `webtelemt_token` |
| **Заказы** | Только свои | Все заказы |
| **Клиенты Telemt** | Только свои профили (read-only) | Полное управление |
| **Secret** | Не показывается | Не показывается |
| **Ссылки / QR** | Да | Да |

Токены **не взаимозаменяемы**: customer JWT не открывает admin API.

## API customer (кратко)

Префикс: `/api/account`. Авторизация: `Authorization: Bearer <customer_token>`.

| Метод | Путь | Auth | Описание |
|-------|------|------|----------|
| `POST` | `/api/account/register` | — | Регистрация → `{ token, email }` |
| `POST` | `/api/account/login` | — | Вход → `{ token, email }` |
| `GET` | `/api/account/me` | customer | `{ id, email }` |
| `GET` | `/api/account/orders` | customer | Список заказов аккаунта |
| `GET` | `/api/account/profiles` | customer | Профили + live Telemt (без secret) |

**Биллинг с привязкой:** `POST /api/billing/orders` с опциональным customer JWT — заказ сразу получает `customer_id`. Без JWT email обязателен в теле запроса.

Rate limit на register/login — как на вход в админку.

Полный список admin/billing endpoints — в [`README.md`](../../README.md) и [`DEV.md`](DEV.md).

## Для администратора: настройка оплаты

### Предварительные условия

1. WebTelemt установлен (`./install.sh`), Telemt API на `127.0.0.1:9091`.
2. Контейнеры: `cd development && docker compose up -d`.
3. Аккаунт [ЮKassa](https://yookassa.ru/).

### Переменные в `development/.env`

```env
YOOKASSA_SHOP_ID=<ID магазина>
YOOKASSA_SECRET_KEY=<секретный ключ>
```

Тариф (при необходимости):

```env
BILLING_PLAN_NAME=Базовый
BILLING_PLAN_PRICE_RUB=299
BILLING_PLAN_PERIOD_DAYS=30
```

Production:

```env
YOOKASSA_RETURN_URL=https://ваш-домен.example
```

Перезапуск: `cd development && docker compose up -d`

Миграция `002_customers.sql` (таблица `customers`) применяется при старте backend.

### Webhook ЮKassa

1. [Личный кабинет ЮKassa](https://yookassa.ru/my) → Настройки → Уведомления.
2. URL: `https://ваш-домен.example/api/billing/webhook/yookassa`
3. Событие: `payment.succeeded`.

### Проверка

| Проверка | Ожидание |
|----------|----------|
| `/buy` | Тариф и форма оплаты |
| `/account/register` | Регистрация покупателя |
| Тестовый платёж | Success + secret; заказ в кабинете |
| Админка → «Заказы» | Заказ `completed`, email, без secret |

### Админ-панель: заказы

ID, дата, сумма, статус, username, email покупателя, флаг просмотра credentials. Secret в админке не отображается.

## Статусы заказа

| Статус | Значение |
|--------|----------|
| `pending` | Ожидает оплаты |
| `paid` | Оплата подтверждена |
| `completed` | Профиль выдан |
| `failed` | Ошибка |
| `expired` | Не оплачен в срок (TTL ~60 мин) |

## FAQ

### Как вернуться к своим профилям после покупки?

Зарегистрируйтесь или войдите в `/account` с тем же email, что при оплате. В кабинете — ссылки подключения и статистика. Secret повторно не выдаётся.

### Покупатель не видит secret после оплаты

1. Обновите `/buy/success?order_id=...` через 10–30 с.
2. Проверьте webhook ЮKassa (HTTPS).
3. Логи: `docker compose -f development/docker-compose.yml logs -f webtelemt`.

### «Учётные данные уже были показаны»

Secret выдаётся один раз. В кабинете доступны ссылки подключения, но не пароль. Новый заказ или помощь администратора.

### Заказ не появился в кабинете

Email при регистрации должен совпадать с email в заказе. Если при покупке email не указывали — привязка невозможна.

### Отправляется ли email с паролем?

Нет. Email только для привязки заказов к кабинету.

### `/buy` — «Тариф недоступен»

Проверьте `YOOKASSA_*`, `DATABASE_URL`, `BILLING_CREDENTIALS_ENCRYPTION_KEY` в `.env` и перезапустите контейнеры.
